Guia basica de Git Y GitHub

Principales comandos de Git

Resumen rápido de los comandos más usados:

  • git init — crear un repositorio local.
  • git clone <url> — clonar un repositorio remoto.
  • git status — ver el estado de los archivos.
  • git add <archivo|.> — añadir cambios al área de preparación (staging).
  • git commit -m "mensaje" — guardar cambios en el historial.
  • git branch / git checkout <rama> — gestionar y cambiar ramas.
  • git merge <rama> — fusionar una rama en la actual.
  • git pull / git push — traer y enviar cambios al remoto.
  • git log — ver el historial de commits.
  • git diff — ver diferencias entre estados.

Ejemplo de flujo básico:

Procesos Detallados con Ejemplos

git init

Inicia un nuevo repositorio:

$ mkdir mi-proyecto
$ cd mi-proyecto
$ git init

git clone

Clona un repositorio existente:

$ git clone https://github.com/usuario/repositorio.git

git add

Añade archivos al staging:

$ git add archivo.txt
$ git add .

git commit

Guarda los cambios en el repositorio:

$ git commit -m "Primera versión del proyecto"

git branch

Crea y gestiona ramas:

$ git branch nueva-funcionalidad
$ git checkout nueva-funcionalidad

git merge

Fusiona ramas:

$ git checkout main
$ git merge nueva-funcionalidad

git pull/push

Sincroniza con repositorio remoto:

$ git pull origin main
$ git push origin main
